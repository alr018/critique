Error Prevention

Using form validation and displaying mandatory fields can help prevent errors from happening.

From the homepage there were two very clear search fields for the user. The first was a prominent “Find your course” search box. After submitting the user was redirected to a page that would either display the search results or indicate that no matches were found. This search field meets the basic requirements for most search fields on the web.

However, the second search field on the homepage, which was unlabeled, did not. Based on placement it can be assumed that the search bar was to do a full site search, but a small hint field in the textbox would have been appreciated. Furthermore, once the user presses submit on the field, they are redirected to a “Google Powered” search page. This did not flow well with the rest of the site. More importantly, this page left the search results completely blank if no results were found. Without an indication of why the page was blank, this can be ambiguous to the user.

In addition to search forms, there were also a few submission forms, such as the “Request a Prospectus” form. This form clearly indicated the mandatory fields and performed validation after pressing submit which indicated which fields were at fault. However, it had an interesting issue where it would only display the email field as having an error until the email was entered correctly. After which, all the other fields would be validated in parallel.

Throughout the website there were standard login pages that required a username and password. These pages performed typical validation after the user pressed submit. They often had helpful text that reminded the user which login they should use. As an example, on the IT login page it specified that the user should enter his IT credentials to login. On the “MyStudentLogin” page it reminded the user that they may need to create an account if it is their first time logging in and had a clear link for registration. This text may help user from struggling to determine why they are unable to login.
